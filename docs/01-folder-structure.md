# AI Powered Restaurant OS — Folder Structure

## Level 1: Root Directory

```
AiPoweredROS/
├── frontend/             # What users see (Next.js app)
├── backend/              # Business logic & database (NestJS + Prisma)
├── ai-service/           # Smart features (Python FastAPI — add later)
├── docker-compose.yml    # Run everything with one command
├── .env                  # All secrets and config in one place
└── README.md             # Project overview
```

---

## Level 2: frontend/ (Next.js)

```
frontend/
├── app/
│   ├── (customer)/       # Pages: /menu, /cart, /orders
│   ├── (waiter)/         # Pages: /waiter/tables, /waiter/orders
│   ├── (kitchen)/        # Pages: /kitchen/tickets
│   ├── (cashier)/        # Pages: /cashier/billing, /cashier/payments
│   └── (admin)/          # Pages: /admin/menu, /admin/employees, /admin/reports
├── components/
│   ├── ui/               # Buttons, inputs, cards, modals
│   ├── layout/           # Sidebar, header, navigation
│   └── orders/           # OrderCard, StatusBadge
├── lib/
│   ├── api.ts            # All backend API calls in one file
│   ├── auth.ts           # Login/logout helpers
│   └── utils.ts          # Format currency, dates
├── package.json
└── next.config.js
```

## Level 3: backend/ (NestJS + Prisma)

```
backend/
├── src/
│   ├── main.ts                    # App bootstrap
│   ├── app.module.ts              # Root module
│   │
│   └── modules/                   # One folder per feature
│       ├── auth/                  # Login, register, JWT tokens
│       ├── users/                 # Employees, roles, permissions
│       ├── menu/                  # Categories, items, prices
│       ├── orders/                # Create, update, track orders
│       ├── kitchen/               # Kitchen Display System
│       ├── billing/               # Invoices, tax calculation
│       ├── inventory/             # Stock tracking
│       ├── payments/              # Stripe/Razorpay integration
│       └── notifications/         # SMS, email, push alerts
├── prisma/
│   ├── schema.prisma              # All database models in ONE file
│   └── migrations/                # Auto-generated by Prisma
├── common/
│   ├── guards/roles.guard.ts      # Role-based access control
│   ├── decorators/                # Custom NestJS decorators
│   └── dto/                       # Reusable data types
├── .env
├── package.json
├── tsconfig.json
└── nest-cli.json
```

## Level 4: ai-service/ (Python FastAPI)

```
ai-service/
├── main.py                # FastAPI app entry point
├── requirements.txt       # Python dependencies
├── routers/
│   ├── predict.py         # POST /predict — demand forecasting
│   └── recommend.py       # POST /recommend — dish suggestions
└── models/
    └── demand_model.pkl   # Trained ML model file
```

## Build Order

| Step | Folder | What You Build |
|------|--------|----------------|
| 1 | backend/ | auth + users + menu modules |
| 2 | backend/ | orders + kitchen + billing modules |
| 3 | frontend/ | Customer menu + cart + checkout |
| 4 | frontend/ | Kitchen display + waiter panel |
| 5 | frontend/ | Admin dashboard + POS cashier |
| 6 | backend/ | Payments + notifications + inventory |
| 7 | ai-service/ | Predictions + recommendations |
